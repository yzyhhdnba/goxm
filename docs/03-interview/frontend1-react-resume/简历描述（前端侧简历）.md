# 简历描述（前端侧简历）

下面给出三版写法，按篇幅从短到长排序。

---

## 版本一：一段式精简版

基于 React + TypeScript + Vite 重构在线视频平台核心前端页面，完成首页推荐、搜索、视频详情、评论回复、个人中心、历史记录、创作者投稿、审核后台与系统通知等模块开发；使用 Redux Toolkit 和 Axios 封装统一状态管理与请求层，完成与 Go 后端的真实接口联调，提升前端工程化程度与业务闭环完整性。

---

## 版本二：校招 / 实习标准版

**PILIPILI Go 并行 React 前端重构项目**  
技术栈：React、TypeScript、Vite、Tailwind CSS、Redux Toolkit、React Router、Axios

- 基于 Vite 搭建 React 前端工程，结合 TypeScript 与 Redux Toolkit 完成项目初始化、全局状态管理和模块拆分
- 对 Axios 进行二次封装，统一处理 token 注入、错误收口与响应解包，提高接口复用性与前后端联调效率
- 重构首页、搜索、视频详情、评论回复、个人中心、历史记录、创作者投稿、审核后台与通知模块，补齐平台核心业务链路
- 实现评论树、回复展开、点赞、来源页透传和分页条件保留等交互细节，优化多页面跳转下的用户体验
- 基于浏览器原生能力实现视频封面截帧功能，并通过持续构建验证保障前端工程稳定性

---

## 版本三：适合面试展开版

**PILIPILI Go 并行 React 前端重构项目**  
在保留原有 Vue 前端的基础上，新建 `frontend1/` 作为并行 React 工程，围绕在线视频平台核心业务完成首页推荐、搜索、视频详情、评论树、个人中心、历史记录、创作者投稿、审核后台和系统通知等模块重构。项目中基于 Redux Toolkit 管理登录态与全局状态，基于 Axios 封装统一请求层，并通过 React Router 实现搜索页、用户主页和视频详情之间的来源页与分页条件保留。重点补齐了评论回复树、历史记录分页、视频投稿、后台审核和通知提醒等交互闭环，使这套 React 前端具备独立构建、联调和面试表达价值。
