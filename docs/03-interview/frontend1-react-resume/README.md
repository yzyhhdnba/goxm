# frontend1 React 前端面试资料

这组文档面向 `frontend1/` 这套并行 React 前端实现，目标是把“做过页面”整理成“可以写进简历、可以稳定讲给面试官听、可以被追问后继续展开”的面试材料。

建议阅读顺序：

1. [前端侧简历.md](./前端侧简历.md)
2. [简历描述（前端侧简历）.md](./简历描述（前端侧简历）.md)
3. [面试整体讲稿.md](./面试整体讲稿.md)
4. [面试可能问题.md](./面试可能问题.md)
5. [前端侧深度技术文档.md](./前端侧深度技术文档.md)

适用场景：

- 校招 / 实习前端岗位
- 偏 React 工程化方向的面试
- 需要把 `frontend1` 从“并行重构练习”讲成“有业务闭环的前端项目”

使用原则：

- 先讲事实，再讲亮点，不要把未实现的能力说成已完成
- 优先强调业务闭环、工程拆分、接口联调、状态管理，而不是空泛背技术名词
- 如果面试官追问后端，可以把话题自然切回 `Go API + React 前端并行联调`
